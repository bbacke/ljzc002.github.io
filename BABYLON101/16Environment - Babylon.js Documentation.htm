Environment

环境

You have come a long way, have learned about shapes, lights, sprites, particles, materials. But there is something missing in your scenes: a proper environment. This is the first of three consecutive How_To that talk about scene environment factors and effects. We will start off with simple scene clearColor (background color), then talk briefly about scene ambientColor, then on to 6-texture skyboxes, and then fog to give an illusion of depth to your scenes.

你已经走了很长的路，学习了关于形状、光源、精灵、粒子、材质的知识。但是在你的场景中还缺少一些东西：一个合适的环境。这篇教程是三篇连续的讨论环境特性和效果的“如何做”教程中的第一篇。我们将从简单的场景背景色开始，然后简短的讨论场景的环境颜色，然后是具有六个纹理的天空盒，接着用雾给你场景添加对深度的模拟。

A picture showing Babylon.js fog in action

一张展示Babylon.js雾效果的图片

How can I do this?

如何做？

We will talk about that nice fog effect, shortly. First, I want to introduce you to two interesting properties on the scene class object:

我们不久将讨论那个很棒的雾效。首先，我希望向你们介绍两个“场景类对象”的有趣属性：

  • scene.clearColor - changes the 'background' color.改变场景的“背景”颜色
  • scene.ambientColor - changes the color used in several effects, including ambient lighting. 改变一些效果中使用的颜色，包括环境光照。

Both of them are very useful, and powerful in their own right.

它们都是很有用的，并且在它们的职权范围内很给力。

Changing the Background color (scene.clearColor)

改变背景颜色（scene.clearColor）

The 'clearColor' property on the scene object is the most rudimentary of environment properties/adjustments. Simply stated, this is how you change the background color of the scene. Here is how it is done:

场景对象的'clearColor'属性是环境属性或者环境选项里最基础的，简单来说，这是你修改场景背景颜色的方法。这里是具体做法：

scene.clearColor = new BABYLON.Color3(0.5, 0.8, 0.5);

Or maybe you want to use one of our preset colors and avoid using the new keyword:

或者你可能想使用我们预定义颜色中的一种，使用预定义颜色可以不使用“new”关键词：

scene.clearColor = BABYLON.Color3.Blue();

This color and property is not used in any calculations for the final colors of mesh, materials, textures, or anything else. It is simply the background color of the scene. Easy.

这个颜色属性对网格、材质、纹理或者其他别的东西的最终颜色计算没有任何影响。它只是场景的背景颜色，就是这么简单。

Changing the Ambient color (scene.ambientColor)

环境颜色的改变（scene.ambientColor）

Conversely, the ambientColor property on the scene object is a very powerful and influential environment property/adjustment. First, let's have a look at its syntax:

相反的，场景对象的环境颜色属性则是一个非常强力并且有很大影响的环境属性或者环境选项。首先，让我们看看这个句法：

scene.ambientColor = new BABYLON.Color3(0.3, 0.3, 0.3);

As you can see, it is set using the same format as clearColor, but ambientColor is used in quite a few calculations toward determining the final colors of scene items.  Mainly, it is used in conjunction with a mesh's StandardMaterial.ambientColor to determine a FINAL ambientColor for the mesh material.

如你所见，它使用和背景颜色相同的格式设置，但是环境颜色在决定场景物体颜色的计算中被用到了很多。总的来讲，它被和网格的StandardMaterial.ambientColor联合使用，来决定网格材质的最终环境颜色。

You will find that when there is no scene.ambientColor, then StandardMaterial.ambientColor and StandardMaterial.ambientTexture will appear to do nothing.  Set a scene.ambientColor of some value, like the example above, and StandardMaterial.ambientColor/StandardMaterial.ambientTexture will become active on meshes where you have applied such.

你将发现，在没有scene.ambientColor设置时，StandardMaterial.ambientColor和StandardMaterial.ambientTexture将没有任何表现。将scene.ambientColor设为某个值，比如上面的例子，则StandardMaterial.ambientColor和StandardMaterial.ambientTexture将会在设置的网格上激活。

By default, scene.ambientColor is set to Color3(0, 0, 0), which means there is no scene.ambientColor.

默认情况下，scene.ambientColor被设为Color3(0, 0, 0)，这意味着不存在环境颜色。

(Please see the section on ambientColors in our Unleash the Standard Material tutorial, for more information.)

（请看我们的“基础材质解析”教程中关于环境颜色的部分，以获取更多信息。）

Skybox

天空盒

To give a perfect illusion of a beautiful sunny sky, we are going to create a simple box, but with a special texture. There is two ways to create a skybox. Let's start with the manual one to understand how things work under the hood and then we will be able to use the automatic one.

为了完美的模拟一个美丽的晴朗天空，我们将建立一个简单的盒子，但是带有一种特殊的纹理。有两种建立天空盒的方法，让我们从手动模式开始，去理解幕后如何工作，然后我们将能够使用自动模式。

Manual creation

手动建立

First, our box, nothing new, just take notice of the disabled backface culling:

首先，我们的盒子，没有什么新的，只是需要注意禁用“背面剔除”：

var skybox = BABYLON.Mesh.CreateBox("skyBox", 100.0, scene);
var skyboxMaterial = new BABYLON.StandardMaterial("skyBox", scene);
skyboxMaterial.backFaceCulling = false;
skyboxMaterial.disableLighting = true;
skybox.material = skyboxMaterial;

Next, we set the infiniteDistance property. This makes the skybox follow our camera's position.

下一步，我们设置无限距离属性。这将使得我们的天空盒跟随相机的位置移动（于是相机永远不会到达天空盒的边缘）

skybox.infiniteDistance = true;

Now we must remove all light reflections on our box (the sun doesn't reflect on the sky!):

现在我们必须移除我们盒子的所有光反射（阳光不会在天空中反射！）

skyboxMaterial.disableLighting = true;

Next, we apply our special sky texture to it. This texture must have been prepared to be a skybox, in a dedicated directory, named “skybox” in our example:

下一步，我们为它设置特殊的天空纹理。这个纹理必须在一个专门的文件目录中做好成为天空盒的准备，在我们的例子里这个目录叫做“skybox”

skyboxMaterial.reflectionTexture = new BABYLON.CubeTexture("textures/skybox", scene);
skyboxMaterial.reflectionTexture.coordinatesMode = BABYLON.Texture.SKYBOX_MODE;

(More about reflectionTextures can be found in our Unleash the Standard Material tutorial.)

（更多关于反射纹理的信息可以在我们的基础材质解析教程里找到。）

In that /skybox directory, we must find 6 sky textures, one for each face of our box. Each image must be named per the corresponding face: “skybox_nx.png”, “skybox_ny.png”, “skybox_nz.png”, “skybox_px.png”, “skybox_py.png”, “skybox_pz.png”.

在这个/skybox目录里，我们必须能找到6个天空纹理，每一个对应我们盒子的一个面。每张图片必须根据对应的面命名：“skybox_nx.png”, “skybox_ny.png”, “skybox_nz.png”, “skybox_px.png”, “skybox_py.png”, “skybox_pz.png”.（p为正方向、n为反方向）

 

Skybox textures need not be textures of sky alone. You can search the Internet for skyboxes and find buildings, hills, mountains, trees, lakes, planets, stars, you name it (all can be used nicely) as part of skybox textures, but some require a payment.

天空盒纹理并不只包括天空的纹理。你可以在网上搜索天空盒，并找到建筑、山丘、山峰、树木、湖泊、行星、星星，所有你能想到的图片都可以很好的作为天空盒纹理的一部分，但是有些需要付费下载。

You can also use dds files to specify your skybox. These special files can contain all information required to setup a cube texture:

你还可以使用dds文件来定制你的天空盒。这个特殊的文件可以包括建立一个立方体纹理所需的全部信息：

skyboxMaterial.reflectionTexture = new BABYLON.CubeTexture("/assets/textures/SpecularHDR.dds", scene);

Final note, if you want your skybox to render behind everything else, set the skybox's renderingGroupId to 0, and every other renderable object's renderingGroupId greater than zero, for example:

最后要注意，如果你希望你的天空盒渲染在其他所有东西的后面（认为天空无限远），就将天空盒的renderingGroupId属性设为0，然后将其他可渲染对象的渲染组Id设为比零大，比如：

skybox.renderingGroupId = 0;
 
// Some other mesh 一些其他网格
myMesh.renderingGroupId = 1;

More info about rendering groups and rendering order can be found here.

更多关于渲染组和渲染顺序的内容可以在这里找到。

Automatic creation

自动建立

Now that we understand how a skybox can be created let's move to a simpler way:

既然我们已经理解了天空盒如何被建立，让我们转向一个更简单的方式

var envTexture = new BABYLON.CubeTexture("/assets/textures/SpecularHDR.dds", scene);
scene.createDefaultSkybox(envTexture, true, 1000);

Fog

雾

Fog is quite an advanced effect, but fog in Babylon.js has been simplified to the maximum. It’s now very easy to add fog to your scenes.  First, we define the fog mode like this:

雾是一种非常高级的效果，但是Babylon.js中雾被最大程度的简化了。现在在你的场景中添加雾是非常简单的。首先，我们这样定义雾的模式：

scene.fogMode = BABYLON.Scene.FOGMODE_EXP;

Here are the available modes:

以下是可用的模式

  • BABYLON.Scene.FOGMODE_NONE - default one, fog is deactivated.默认模式，雾是不起作用的。
  • BABYLON.Scene.FOGMODE_EXP - the fog density is following an exponential function. 雾的浓度按指数函数变化
  • BABYLON.Scene.FOGMODE_EXP2 - same that above but faster. 和上一种相同，但快一些
  • BABYLON.Scene.FOGMODE_LINEAR - the fog density is following a linear function. 雾的浓度按线性函数变化。

-> If you choose the EXP, or EXP2 mode, then you can define the density option (default is 0.1):

如果你选择EXP或者EXP2模式，那么你可以定义浓度选项（默认为0.1）：

scene.fogDensity = 0.01;

-> Otherwise, if you choose LINEAR mode, then you can define where fog starts and where fog ends:

否则，如果你选择线性模式，那么你可以定义雾的起止位置

scene.fogStart = 20.0;
scene.fogEnd = 60.0;

Finally, whatever the mode, you can specify the color of the fog (default is BABYLON.Color3(0.2, 0.2, 0.3)):

最后，不论哪种模式，你都可以设定雾的颜色（默认为BABYLON.Color3(0.2, 0.2, 0.3)）：

scene.fogColor = new BABYLON.Color3(0.9, 0.9, 0.85);

See, we told you it was easy.

看到了吧，我们告诉过你它很简单。
