Cameras

相机

Of the many cameras available in Babylon.js the two most used are probably - the Universal Camera used for First Person Movement and the Arc Rotate Camera which is an orbital camera. Though with the advent of WebVR this may change.

在Babylon.js可用的许多种相机中，最常用的可能是这两种——在第一人称移动时使用的通用世界相机和作为轨道相机的圆弧旋转相机。而随着WebVR的到来这一切可能会变化。

For input control by the user all cameras need to be attached to the canvas once constructed using

为了响应用户的控制输入，所有的相机在构造完成之后需要和canvas绑定起来。

camera.attachControl(canvas, true);

The second parameter is optional and defaults to false. When false then default actions on a canvas event are prevented. Set to true to allow canvas default actions.

第二个参数是可选的，其默认值为false。当设为false时canvas的默认事件将被阻止。设为true则允许canvas的默认事件。

Notes

注意

  1. A Gamepad may be used a controller. 游戏手柄相机可能需要专用的控制器。
  2. For touch control either PEP or hand.js is needed. 要进行触控操作，PEP或hand.js库是必须的。

Universal Camera

通用世界相机

This was introduced with version 2.3 of Babylon.js and is controlled by the keyboard, mouse, touch or gamepad depending on the input device used, with no need for the controller to be specified. This extends and replaces the Free Camera, the Touch Camera and the Gamepad Camera which are all still available.

这种相机在Babylon.js的2.3版引入，可由键盘鼠标、触控或者游戏手柄等设备控制，不需要明确的指定控制器的类型。这一相机扩展并代替了“自由相机”、“触控相机”、“游戏手柄相机”（但这三种相机仍然可用）。

The Universal Camera is now the default camera used by Babylon.js if nothing is specified, and it’s your best choice if you’d like to have a FPS-like control in your scene. All demos on babylonjs.com are based upon that feature. Plug a Xbox controller into your PC and using it you’ll still be able to navigate most of the demos.

现在通用世界相机是Babylon.js在没有特殊设置时默认使用的相机，同时也是你希望在场景中使用类似FPS的控制方式时的最佳选择。所有babylonjs.com网站上的例子都以这个特性为基础。将一个Xbox控制器插到你的PC上使用时，你仍然能使用绝大部分的示例程序。

The default actions are:

默认的动作是：

1.      Keyboard - The left and right arrows move the camera left and right, and up and down arrows move it forwards and backwards;

键盘-左键和右键左右移动相机，上键和下键前后移动相机

2.      Mouse - Rotates the camera about the axes with the camera as origin;

鼠标-以相机为原点旋转相机

3.      Touch - Swipe left and right to move camera left and right, and swipe up and down to move it forward and backwards;

触控-左右滑动左右移动相机，上下滑动前后移动相机

4.      gamepad - corresponds to device.

游戏手柄-和具体设备有关。

Note 注意

  • Using keys in the Playground requires you to click inside the rendering area to give it the focus.

要在Playground中使用键盘控制，需要先点入渲染区域（canvas）以使之获得焦点。

Constructing a Universal Camera

构造通用世界相机

 
// Parameters : name, position, scene 参数：名字、位置、场景对象
    var camera = new BABYLON.UniversalCamera("UniversalCamera", new BABYLON.Vector3(0, 0, -10), scene);
 
// Targets the camera to a particular position. In this case the scene origin 将相机指向一个方向，这里是指向场景的原点
    camera.setTarget(BABYLON.Vector3.Zero());
 
// Attach the camera to the canvas 将相机和canvas绑定起来
    camera.attachControl(canvas, true);

一个通用世界相机的例子

（Universal可以翻译为"世界"也可以翻译为“通用”，考虑到这种相机既能够在虚拟世界中自由移动，又具备兼容多种控制设备的通用性，将其译为“通用世界相机”）

Arc Rotate Camera

圆弧旋转相机

This camera always points towards a given target position and can be rotated around that target with the target as the center of rotation. It can be controlled with cursors and mouse, or with touch events.

这个相机永远指向一个指定的位置，并且可以以目标为中心，绕着目标旋转。可以通过游标（上下左右键）和鼠标或者触摸事件控制它。

Think of this camera as one orbiting its target position, or more imaginatively as a spy satellite orbiting the earth. Its position relative to the target (earth) can be set by three parameters, alpha (radians) the longitudinal rotation, beta (radians) the latitudinal rotation and radius the distance from the target position. Here is an illustration:

想象这个相机绕着它的目标做轨道运动，或者更有画面感的——一颗间谍卫星绕着地球运动。它相对于目标（地球）的位置可以通过三个参数设置：alpha（弧度值）表示旋转的经度，beta（弧度值）表示旋转的纬度，radius表示距离目标的距离。下面是示意图：

Setting beta to 0 or PI can, for technical reasons, cause problems and in this situation beta is offset by 0.1 radians (about 0.6 degrees).

将beta设为0或者PI会由于技术原因导致程序错误，在这种情况下beta将被补偿0.1弧度（大约是6度）

Both alpha and beta increase in a clockwise direction.

alpha和beta都是沿顺时针方向增加的。

The position of the camera can also be set from a vector which will override any current value for alpha, beta and radius. This can be much easier than calculating the required angles.

相机的位置同样可以通过一个向量来设定，这会覆盖所有已有的alpha、beta、radius值。与计算所需要的角度相比这样做可能更简单。

Whether using the keyboard, mouse or touch swipes left right directions change alpha and up down directions change beta.

不管是用键盘鼠标还是触控滑动，左右改变alpha上下改变beta。

Constructing an Arc Rotate Camera

构造圆弧旋转相机

 
// Parameters: alpha, beta, radius, target position, scene 参数：阿尔法、贝塔、半径、目标位置、场景
    var camera = new BABYLON.ArcRotateCamera("Camera", 0, 0, 10, new BABYLON.Vector3(0, 0, 0), scene);
 
// Positions the camera overwriting alpha, beta, radius 设置相机的位置覆盖alpha、beta、radius
    camera.setPosition(new BABYLON.Vector3(0, 0, 20));
 
// This attaches the camera to the canvas
    camera.attachControl(canvas, true);

一个圆弧旋转相机的例子

 

Panning with an ArcRotateCamera is also possible by using CTRL + MouseLeftClick, the default action. You can specify to use MouseRightClick instead, by setting useCtrlForPanning to false in the attachControl call :

默认情况下，也可以用ctrl+鼠标左键单击来旋转圆弧旋转相机。你也可以设定用鼠标右键单击来代替这一操作，通过在调用attachControl时设置useCtrlForPanning属性为false

FollowCamera

跟随相机

The Follow Camera does what it says on the tin. Give it a mesh as a target and from whatever position it is currently at it will move to a goal position from which to view the target. When the target moves so will the Follow Camera.

跟随相机正如它所说的那样。给它一个网格作为目标，则不论它目前在哪里，他都将移动到一个目标位置，然后从那里盯着目标网格。当目标移动时跟随相机也会移动。

The initial position of the Follow Camera is set when it is created then the goal position is set with three parameters:

跟随相机的初始化位置在建立时设置，目标位置通过以下三个参数设置：

1.      the distance from the target - camera.radius; 距目标网格的距离

2.      the height above the target - camera.heightOffset; 比目标网格高出多少

3.      the angle in degrees around the target in the x y plane. 在xy平面中相机绕目标网格转过的角度（值用角度表示）

The speed with which the camera moves to a goal position is set through its acceleration (camera.cameraAcceleration) up to a maximum speed (camera.maxCameraSpeed).

相机移动到目标位置的速度通过它的“加速度”属性设置(camera.cameraAcceleration)，并有一个最大值（camera.maxCameraSpeed）

（也许可以用这种相机作为第三人称游戏的越肩视角，这时“目标网格target”是角色的中心而“目标位置goal”是角色的肩部，但这一相机似乎并不像前两种相机支持使用鼠标改变视角，需要自己编写一套监听旋转的方法）

Constructing a Follow Camera

构造一个跟随相机

 
// Parameters: name, position, scene
var camera = new BABYLON.FollowCamera("FollowCam", new BABYLON.Vector3(0, 10, -10), scene);
 
// The goal distance of camera from target 相机的目标位置距目标网格的距离
camera.radius = 30;
 
// The goal height of camera above local origin (centre) of target 相机的目标位置相对于目标网格的局部坐标系原点（中心点）的高度
camera.heightOffset = 10;
 
// The goal rotation of camera around local origin (centre) of target in x y plane 相机的目标位置在xy平面中绕目标网格的原点转过的角度
camera.rotationOffset = 0;
 
// Acceleration of camera in moving from current to goal position 相机从当前位置移动到目标位置的加速度
camera.cameraAcceleration = 0.005
 
// The speed at which acceleration is halted 最高速度
camera.maxCameraSpeed = 10
 
// This attaches the camera to the canvas
camera.attachControl(canvas, true);
 
// NOTE:: SET CAMERA TARGET AFTER THE TARGET'S CREATION AND NOTE CHANGE FROM BABYLONJS V 2.5
注意：：要在建立了目标网格对象之后再设置相机的目标网格属性，并且注意从Babylon.js2.5版开始的变化。
// targetMesh created here. 在这里建立目标网格
camera.target = targetMesh;   // version 2.4 and earlier 2.4版和之前
camera.lockedTarget = targetMesh; //version 2.5 onwards 从2.5版开始

A Playground Example of a Follow Camera following a moving target -

一个跟随相机跟随移动的目标网格的例子

var createScene = function () {

 

    // This creates a basic Babylon Scene object (non-mesh)

    var scene = new BABYLON.Scene(engine);

 

    /********** FOLLOW CAMERA EXAMPLE **************************/

 

    // This creates and initially positions a follow camera     

    var camera = new BABYLON.FollowCamera("FollowCam", new BABYLON.Vector3(0, 10, -10), scene);

    

    //The goal distance of camera from target

    camera.radius = 30;

    

    // The goal height of camera above local origin (centre) of target

    camera.heightOffset = 10;

    

    // The goal rotation of camera around local origin (centre) of target in x y plane

    camera.rotationOffset = 0;

    

    //Acceleration of camera in moving from current to goal position

    camera.cameraAcceleration = 0.005

    

    //The speed at which acceleration is halted

    camera.maxCameraSpeed = 10

    

    //camera.target is set after the target's creation

   

    // This attaches the camera to the canvas

    camera.attachControl(canvas, true);

    

    /**************************************************************/

 

    // This creates a light, aiming 0,1,0 - to the sky (non-mesh)

    var light = new BABYLON.HemisphericLight("light", new BABYLON.Vector3(0, 1, 0), scene);

    

    //Material

    var mat = new BABYLON.StandardMaterial("mat1", scene);

    mat.alpha = 1.0;

    mat.diffuseColor = new BABYLON.Color3(0.5, 0.5, 1.0);

    var texture = new BABYLON.Texture("http://jerome.bousquie.fr/BJS/images/spriteAtlas.png", scene);

    mat.diffuseTexture = texture;

    

    //Different face for each side of box to show camera rotation

    var hSpriteNb =  6;  // 6 sprites per row

    var vSpriteNb =  4;  // 4 sprite rows   

    

    var faceUV = new Array(6);

 

    for (var i = 0; i < 6; i++) {

        faceUV[i] = new BABYLON.Vector4(i/hSpriteNb, 0, (i+1)/hSpriteNb, 1 / vSpriteNb);

    }

     

    // Shape to follow

    var box = BABYLON.MeshBuilder.CreateBox("box", {size: 2, faceUV: faceUV }, scene);

    box.position = new BABYLON.Vector3(20, 0, 10);

    box.material = mat;

 

    //create solid particle system of stationery grey boxes to show movement of box and camera

    var boxesSPS = new BABYLON.SolidParticleSystem("boxes", scene, {updatable: false});

   

    //function to position of grey boxes

    var set_boxes = function(particle, i, s) {

        particle.position = new BABYLON.Vector3(-50 + Math.random()*100, -50 + Math.random()*100, -50 + Math.random()*100);

    }

   

    //add 400 boxes

    boxesSPS.addShape(box, 400, {positionFunction:set_boxes}); 

    var boxes = boxesSPS.buildMesh(); // mesh of boxes

 

    /*****************SET TARGET FOR CAMERA************************/

    camera.lockedTarget = box;

    /**************************************************************/

    

    

    //box movement variables

    var alpha = 0;

    var orbit_radius = 20

    

    

    //Move the box to see that the camera follows it    

    scene.registerBeforeRender(function () {

     alpha +=0.01;

     box.position.x = orbit_radius*Math.cos(alpha);

     box.position.y = orbit_radius*Math.sin(alpha);

     box.position.z = 10*Math.sin(2*alpha);

    

     //change the viewing angle of the camera as it follows the box

     camera.rotationOffset = (18*alpha)%360;

    });

    

    return scene;

 

};

 

AnaglyphCameras

浮雕相机

These extend the use of the Universal and Arc Rotate Cameras for use with red and cyan 3D glasses. They use post-processing filtering techniques.

这些相机扩展了通用世界相机和圆弧旋转相机以支持红蓝3D眼镜。它们使用后期效果滤镜技术。

Constructing Anaglyph Universal Camera

构造通用世界浮雕相机

 
// Parameters : name, position, eyeSpace, scene 参数：名字、位置、眼睛空间，场景
var camera = new BABYLON.AnaglyphUniversalCamera("af_cam", new BABYLON.Vector3(0, 1, -15), 0.033, scene);

Constructing Anaglyph ArcRotateCamera

构造圆弧旋转浮雕相机

 
// Parameters : name, alpha, beta, radius, target, eyeSpace, scene
var camera = new BABYLON.AnaglyphArcRotateCamera("aar_cam", -Math.PI/2, Math.PI/4, 20, new BABYLON.Vector3.Zero(), 0.033, scene);

The eyeSpace parameter sets the amount of shift between the left eye view and the right eye view. Once you are wearing your 3D glasses, you might want to experiment with this float value.

这里的“眼睛空间”参数定义了左眼视角和右眼视角之间的距离。在你带着你的3D眼镜时，你可能想要体验这个浮点数的作用。

You can learn all about anaglyphs by visiting a Wikipedia page that explains it thoroughly.

你可以学习更多和“浮雕”有关的知识，通过访问这个解释的很透彻的维基网页。

Device Orientation Camera

设备朝向相机

This is a camera specifically designed to react to device orientation events such as a modern mobile device being tilted forward or back and left or right.

这是一个专门设计用来响应设备朝向变化事件的相机，比如一个现代移动设备（手机）前后左右倾斜。

Virtual Joysticks Camera

虚拟摇杆相机

This is specifically designed to react to Virtual Joystick events. Virtual Joysticks are on-screen 2D graphics that are used to control the camera or other scene items.

这个相机被特别设计来响应虚拟摇杆动作。虚拟摇杆是位于2D屏幕上的图片，被用来控制相机或者场景中的其他对象。（比如王者荣耀、手游吃鸡）

Complete sample

完整例子

Here is a complete sample that loads the Espilit demo and switches the default camera to a virtual joysticks camera:

这是一个完整的例子，它加载了Espilit示例并且把默认的相机换成了虚拟摇杆相机

document.addEventListener("DOMContentLoaded", startGame, false);
function startGame() {
  if (BABYLON.Engine.isSupported()) {
    var canvas = document.getElementById("renderCanvas");
    var engine = new BABYLON.Engine(canvas, true);
 
    BABYLON.SceneLoader.Load("Espilit/", "Espilit.babylon", engine, function (newScene) {
 
      var VJC = new BABYLON.VirtualJoysticksCamera("VJC", newScene.activeCamera.position, newScene);//从文件中加载一个设定完毕的场景
      VJC.rotation = newScene.activeCamera.rotation;//让虚拟摇杆相机继承场景中已有相机的属性
      VJC.checkCollisions = newScene.activeCamera.checkCollisions;
      VJC.applyGravity = newScene.activeCamera.applyGravity;
 
      // Wait for textures and shaders to be ready 等待纹理和着色器准备就绪
      newScene.executeWhenReady(function () {
        newScene.activeCamera = VJC;//将虚拟摇杆相机设为“激活相机”
        // Attach camera to canvas inputs 绑定相机和canvas
        newScene.activeCamera.attachControl(canvas);
        // Once the scene is loaded, just register a render loop to render it 在场景加载完毕后，注册一个渲染循环去渲染它
        engine.runRenderLoop(function () {
          newScene.render();
        }),
      }),
    }, function (progress) {
    // To do: give progress feedback to user.把执行回调交给使用者
    }),
  }
}

If you switch back to another camera, don’t forget to call the dispose() function first. Indeed, the VirtualJoysticks are creating a 2D canvas on top of the 3D WebGL canvas to draw the joysticks with cyan and yellow circles. If you forget to call the dispose() function, the 2D canvas will remain, and will continue to use touch events input.

如果你切换回另一个相机，不要忘了先调用dispose()方法。事实上虚拟摇杆功能在3DWebGL canvas之上又建立了一个2D的canvas，并在这个2Dcanvas上绘制青色和黄色的圈作为摇杆。如果你忘了调用dispose()方法，这个2Dcanvas将仍然存在，并继续接收触摸事件的输入。

FlyCamera

飞行相机（可以用来模拟飞机）

This camera imitates free movement in 3D space, think "a ghost in space." It comes with an option to gradually correct Roll, and also an option to mimic banked-turns.

这种相机模仿3D空间中的运动，想象一下“空间中的幽灵”。它有一个逐步纠正滚动（也就是机体偏斜之后自动回正）的选项，同时还有一个模拟转弯时机体倾斜的选项

。Its defaults are:

它的默认控制为：

1.      Keyboard - The A and D keys move the camera left and right. The W and S keys move it forward and backward. The E and Q keys move it up and down.

键盘-AD键控制相机左右运动。WS键前后移动相机，EQ键上下移动

2.      Mouse - Rotates the camera about the Pitch and Yaw (X, Y) axes with the camera as origin. Holding the right mouse-button rotates the camera about the Roll (Z) axis with the camera as origin.

鼠标-以相机为原点绕俯仰轴和偏航轴（X轴Y轴）旋转相机。按住鼠标右键，则可以相机为原点绕滚动轴（Z轴）旋转相机

Constructing a Fly Camera

构造一个飞行相机

 
// Parameters: name, position, scene
var camera = new BABYLON.FlyCamera("FlyCamera", new BABYLON.Vector3(0, 5, -10), scene);
 
// Airplane like rotation, with faster roll correction and banked-turns.
喜欢翻转的飞行器，会带有更快的旋转修正和转弯偏斜。
// Default is 100. A higher number means slower correction.
默认值为100，更高的值意味着更慢的回正
camera.rollCorrect = 10;
// Default is false.默认为没有转弯偏斜
camera.bankedTurn = true;
// Defaults to 90° in radians in how far banking will roll the camera. 转弯偏斜的极限，默认为90度，用弧度制表示，
camera.bankedTurnLimit = Math.PI / 2;
// How much of the Yawing (turning) will affect the Rolling (banked-turn.) 偏航对旋转偏斜的影响程度
// Less than 1 will reduce the Rolling, and more than 1 will increase it. 比1低将降低偏斜程度，高于1将增强它
camera.bankedTurnMultiplier = 1;
 
// This attaches the camera to the canvas 
camera.attachControl(canvas, true);

Customizing inputs

自定义输入

The cameras rely upon user inputs to move the camera. If you are happy with the camera presets Babylon.js is giving you, just stick with it.

相机依靠用户的输入来移动相机。如果你使用Babylon.js提供给你的默认设置使用的很开心，坚持使用就好。

If you want to change user inputs based upon user preferences, customize one of the existing presets, or use your own input mechanisms. Those cameras have an input manager that is designed for those advanced scenarios. Read customizing camera inputs to learn more about tweaking inputs on your cameras.

如果你想按照用户的偏好来修改用户输入，来自定一个已有的预设，或者使用你自己的输入机制。这些相机都具有一个输入管理器，这个管理器专门针对这些高级方案设计。阅读“自定义相机输入”来学习如何修改相机的输入。
